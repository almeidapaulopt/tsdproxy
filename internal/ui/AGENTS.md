# internal/ui

Server-rendered HTML via templ (`*.templ` source → `*_templ.go` generated). Components, pages, layouts for the dashboard.

## STRUCTURE

| File | Role |
|------|------|
| `pages/dashboard.templ` | Top-level page shell — navbar, search input, filter/sort/group controls, SSE mount (`<div hx-sse:connect="/stream">`), toasts. Defines `DashboardData`, `DashboardView`, `ProxyViewItem`, `DashboardGroup`. Where to add new top-level pages. |
| `pages/proxylist.templ` | All proxy UI — `ProxyCard`, `ProxyCardInner`, `ProxyModal`, `ProxyModalContent`, `ActionsPanel`, `infoPanel`, `logsPanel`, `LogLine`/`LogLines`/`LogPlaceholder`, `ModalStatusBadge`, `DomainStatusBadges`, `ProxyListRow`, `ProxyListFragment`. Internal helpers: `proxyItem`, `proxyListContent`, `modalTabs`, `copyButton`. Defines `ProxyData`, `PortEntry`, `StatusHistoryEntry`. |
| `pages/forbidden.templ` | 403 page (`ForbiddenPage(message)`). |
| `pages/*_templ.go` | GENERATED from `.templ` sources — DO NOT EDIT. |
| `pages/pages.go` | `//go:generate templ generate` stub. |
| `ui.go` | `RenderTempl` — single render entrypoint used by all handlers. |
| `components/components.go` | `IconURL`, `InlineIcon` — wraps `web.IconSVG`. |
| `layouts/layouts.go` | Empty placeholder package (no layouts yet). |
| `static/.keep` | Empty directory marker. |

## CRITICAL RULES

- **`.templ` is source; `_templ.go` is GENERATED.** Never hand-edit `_templ.go`. Generated files start with `// Code generated by templ - DO NOT EDIT.`.
- **Regenerate after editing `.templ`**: `make dev` runs air with templ watcher; or run `templ generate ./...` manually. CI fails if generated files are out of sync.
- **Imports inside `.templ` files** follow project ordering: stdlib → third-party → `github.com/almeidapaulopt/tsdproxy`. No new template dependencies without reason.

## RENDERING

- `ui.RenderTempl(w, r, cmp)` is the single render entrypoint. Sets `Content-Type: text/html; charset=utf-8`, renders into a `bytes.Buffer`, then `WriteTo(w)`. Returns error from `cmp.Render(r.Context(), &buf)`.
- Handlers in `internal/dashboard/dash.go` call `ui.RenderTempl(w, r, pages.Dashboard(viewData))`, `pages.ProxyListFragment(viewData)`, `pages.ProxyModal(data)`, `pages.ActionsPanel(data)`.
- SSE HTML fragments use the dashboard package's own `ssePartialElement` (see `internal/dashboard/sse.templ`), not this package.

## DATA CONTRACT

- `pages.Dashboard(d pages.DashboardData)` — `DashboardData{ Prefs model.Preferences; Proxies DashboardView; User model.Whois; Version string; Search string; IsAdmin bool }`. Built by `dashboard.buildDashboardViewData()`.
- `pages.ProxyCard(item pages.ProxyData)` is the per-proxy render contract — fields drive icon, label, status badge, port entries, action buttons, health dot, domain badges. Built by `dashboard.buildProxyDataFromProxy()`.
- Any new field on the dashboard must be added to `pages.ProxyData` (or `pages.DashboardData`) AND populated by the dashboard builder AND consumed by the templ template that renders it.

## ICON HELPERS

- `components.IconURL(name)` returns the `/icons/` URL path for an icon. Preserves explicit extensions (`.png`, `.webp`), appends `.svg` otherwise. Empty name → `/icons/tsdproxy.svg`.
- `components.IconImg(name, class)` renders an `<img>` tag for proxy icons. The browser drives the HTTP request, which triggers on-demand download via the `/icons/` handler. Use for proxy icons (user-facing container icons).
- `components.InlineIcon(name, class)` renders an inline `<svg>` for chrome glyphs that need `fill="currentColor"` theming (copy/star/info). Reads from cache or returns embedded default. Does NOT trigger downloads — used synchronously during templ render.
- Proxy cards now use `IconImg` (on-demand via `<img>`). Chrome glyphs stay as `InlineIcon`.

## GOTCHAS

- **Regenerate templates before committing**: CI runs `templ generate` and fails if generated files are out of sync. Local `make dev` auto-regenerates via air.
- **`hx-swap="outerHTML"` on proxy actions**: every action button in `ActionsPanel` targets `#actions-panel-{dom.SafeID(item.Name)}` and the handler returns a freshly rendered `ActionsPanel`. This is why `proxyActionHandler` re-renders `pages.ActionsPanel(...)` on HX-Request.
- **`ActionsPanel` button visibility is status-gated**: Restart shows for Running/Paused/Error/Stopped; Pause for Running; Resume for Paused; Re-authenticate for Authenticating/AwaitingApproval; no buttons while Initializing/Starting/Stopping. Preserve these gates when editing.
- **`hx-swap="none"`** is used elsewhere in `dashboard.templ` (theme toggle, dark-mode checkbox) — but only for SSE-driven refresh cases, never for proxy actions.
- **`dom.SafeID(name)`**: HTML id-safe encoding of proxy name. Use in all `#actions-panel-{SafeID(name)}`, `#log-lines-{SafeID(name)}`, `#modal-status-{SafeID(name)}` targeting patterns.
- **No layouts package yet**: `layouts/layouts.go` is a 4-line placeholder. If adding shared layout (header/footer), start here.
- **Do not import `web` directly in templates** except via the `components` wrapper — keeps the icon abstraction in one place and avoids circular deps.
- **`ProxyData` lives in `pages`, not `dashboard`**: the dashboard package imports `pages.ProxyData`. Don't move it without updating `buildProxyDataFromProxy`.
